For my Game project using a Grid

I will be creating my own version of the game Sudoku!

This version will use the MVC design pattern and will contain a Model and a View

  • The Model contains information about the game-play, state of the game, the parameters and objects
    (game pieces) of the game.
    • For the case of Sudoku, I will have cell objects, interactive buttons, and a drop-down menu.
    • Each object will contain parameters as needed for the gameplay.
    • The "game pieces" will be numbers contained within the cells of a table.
    • The Model for this game will contain the information for all of these things.
  • The View contains functions, parameters and information about how the game is displayed.
    • The Controller will be contained within the View.
    • "For this assignment, don't worry about separating the controller from the view."
    • In a nutshell, the view produces the game interface, or game-board design.

Playing this Version of Sudoku

  • The player will see a blank game-grid and other elements of the game's interface
  • The player must then select a difficulty from the drop-down list of difficulties and a new puzzle of that difficulty will auto-load on selection.
  • A certain amount of the blank cells on the game-grid will be filled in automatically.
  • The player can select any empty-cells on the game-grid and possible correct answers for that cell will be given in a separate clickable grid called the keypad.
  • The player clicks a check answers to check their answers.
  • If all the answers are correct a message will say "everything is good so far", or if there is a conflict the conflicting cells will be highlighted in red".
  • Once all of the numbers have been filled in and all of the answers are correct the game is over and a "you win" message will display.

  • Winning the game:

  • In order to win at Sudoku you must enter the numbers 1 through 9 in each row, column, and mini-grid (3 x 3 table
    within the larger table making up the entire game board). You may enter the numbers in any order but you cannot
    repeat the same number twice in a given row, column, or mini-grid. For example, if there is already a 6 in a given row,
    then there must not be another 6 in the same horizontal row (across the entire board), vertical column (across the entire board)
    or in the same 3 x 3 mini-grid, or else one of the sixes must be in the wrong place...this is called a "conflict"!!!
    Once you have filled in the entire Sudoku Game Board with numbers 1 - 9, having no conflicts, YOU WIN!
